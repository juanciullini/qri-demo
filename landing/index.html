Flujo de Pago QR Interoperable

El ciclo completo de un pago QR Transferencias 3.0, desde el escaneo hasta la acreditación, procesado en menos de 15 segundos.

Paso 1

Escaneo del QR

El comprador escanea el QR EMVCo del comercio con su billetera digital. El QR contiene los datos del comercio en formato TLV.

Paso 2

Intención de Pago

La billetera envía la intención de pago a Coelsa. Se valida el comercio, monto y datos. Coelsa confirma la operación.

Paso 3

Débito y Crédito

Se debita la cuenta del pagador vía DEBIN y se acredita al comercio vía CVU. Todo en tiempo real con confirmación instantánea.

Paso 4

Acreditación

Fondos acreditados en la cuenta del comercio. Liquidación automática con comisiones configurables por perfil de comercio.

Ejemplo: Flujo de integración completo

1
Autenticarse — Tu backend obtiene un JWT llamando a POST /api/auth/login
2
Dar de alta un comercio — Enviás CUIT, CBU, MCC y datos del comercio. QRi lo registra en Coelsa.
3
Generar QR — Pedís un QR estático (monto libre) o dinámico (monto fijo, con vencimiento). Recibís el string EMVCo TLV listo para mostrar.
4
Procesar pagos — Cuando un usuario escanea el QR, Coelsa notifica a QRi vía webhooks (intención, débito, acreditación). La transacción avanza automáticamente.
5
Consultar estado — Polleas GET /api/transactions o usás WebSocket para actualizaciones en tiempo real.
6
Liquidar — Generás liquidaciones por período. El sistema calcula comisiones, neto a pagar y exporta para conciliación.

Ejemplo de código

integration.ts 
// 1. Autenticación
const { access_token } = await fetch('/api/auth/login', {
  method: 'POST',
  body: JSON.stringify({
    email: 'admin@qri.app',
    password: 'Admin2026$'
  })
}).then(r => r.json())

// 2. Crear comercio
const merchant = await fetch('/api/merchants', {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${access_token}`
  },
  body: JSON.stringify({
    business_name: 'Cafetería Test',
    cuit: '30712345671',
    cbu:  '0070000000000000000010',
    mcc_codes: [{
      mcc: '5812',
      desc: 'Restaurantes',
      commission: 2.5
    }]
  })
}).then(r => r.json())

// 3. Generar QR dinámico
const qr = await fetch('/api/qr/dynamic', {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${access_token}`
  },
  body: JSON.stringify({
    merchant_id: merchant.id,
    amount: 1500.50,
    expiration: 30
  })
}).then(r => r.json())

// qr.qr_data → string EMVCo TLV listo
// qr.qr_id_trx → ID para tracking
